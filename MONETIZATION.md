# Monetization Strategy

## Goal

Keep Aura visible and adoptable as an open product while preserving the parts that create defensible commercial value.

Recommended model:

`open-core + paid cognitive extensions + commercial licensing`

## Why This Model Fits Aura

- The project already benefits from public adoption, downloads, and stars.
- Full secrecy would reduce discoverability and trust.
- Full disclosure of the strongest internals would weaken the moat.
- Aura's advantage is not only code, but also tuning, validation discipline, and productized cognitive behavior.

## Open vs Paid Split

### Open / Community

Keep public:

- core SDK surface
- local storage and recall baseline
- basic maintenance lifecycle
- integration examples
- Python bindings for baseline usage
- documentation and onboarding
- stable public inspection APIs that drive adoption

Community goal:

- easy trial
- strong GitHub visibility
- broad embeddability

### Paid / Commercial

Keep proprietary or commercially licensed:

- advanced cognitive tuning and thresholds
- hardened rerank logic and premium configurations
- premium evaluation harnesses and campaign infrastructure
- advanced monitoring packs and operator tooling
- enterprise support, deployment guidance, and custom tuning
- hosted or licensed premium cognitive modules

Commercial goal:

- preserve differentiation
- create licenseable value beyond the free SDK

## Recommended Product Tiers

### Aura Community

- local memory engine
- baseline recall
- public docs and examples
- community support

### Aura Pro

- advanced cognitive features
- premium monitoring and evaluation tooling
- commercial support
- enterprise deployment help
- custom tuning / licensing

### Aura Enterprise

- private licensing
- custom integration
- internal deployment rights
- premium SLA / support
- partnership or OEM terms

## Practical Moat

Do not rely on code secrecy alone.

Aura's moat should come from:

- proprietary tuning
- faster iteration
- stronger eval discipline
- brand and trademark
- commercial support
- premium packaging of the best internals

## Release Strategy

Recommended release pattern:

1. Openly release visible product improvements that increase adoption.
2. Keep the most valuable tuning, operator tooling, and premium packaging under commercial control.
3. Document the commercial path clearly in public docs.

## What To Avoid

- Shipping the full premium moat in the public repo and hoping licenses alone will protect it.
- Hiding the entire project and losing distribution.
- Letting product messaging imply that all internals are free if that is not the commercial intent.

## Immediate Recommendation

Aura's current state supports this split:

- public product story: local cognitive memory engine
- commercial moat: hardened higher-order tuning, premium eval/ops workflows, enterprise licensing

The safest near-term move is to keep the project visible, ship the stable public surface, and commercialize the differentiating operational and cognitive premium layer.
